# 验证检查清单

> 用于角色交接时的完成前验证。每个角色在输出交付物前，必须逐项检查。

---

## 通用检查

- [ ] 所有 `@handoff` 标签已更新为最新状态
- [ ] 所有 `TODO/FIXME/HACK` 已更新
- [ ] 已知问题清单已更新
- [ ] 未做决策已记录 `@pending-decisions`
- [ ] 交接条件已满足
- [ ] 文件命名符合 `deliverables.md` 规范

## 阶段0：构思

- [ ] 需求已收敛为可执行的 MVP 范围
- [ ] 目标用户已定义
- [ ] 核心差异化已识别

## 阶段1：规划

- [ ] PRD 已签收（Producer + Architect）
- [ ] ADR 已完成关键技术决策
- [ ] 执行计划已拆解为原子任务
- [ ] 每个任务有明确完成标准(DoD)
- [ ] 启动会已召开

## 阶段2：设计

- [ ] 所有公式有推导过程
- [ ] 所有数值表有 min/max 边界
- [ ] 系统耦合已显式标注
- [ ] 边界条件已覆盖（零值/负值/溢出）
- [ ] 不确定项已标 TODO

## 阶段3：交互+视觉

- [ ] 所有界面状态已覆盖（正常/空/加载/错误/禁用/极限）
- [ ] 色彩与世界观一致
- [ ] 可点击区域 ≥44px
- [ ] 字体层级不超过 4 级
- [ ] Developer 确认技术可实现
- [ ] 动画方案已标注性能预算

## 阶段4：开发 TDD

- [ ] 所有模块暴露 init/destroy 接口
- [ ] destroy 中解绑所有事件和定时器
- [ ] 所有外部输入有空值保护
- [ ] 所有 DOM 查询有返回值检查
- [ ] Magic Number 已替换为命名常量
- [ ] TODO 格式为 `// TODO(role): description`
- [ ] 代码审查通过
- [ ] 遵循 `AI编程规范.md` 全部要求

## 阶段5：质量审计

- [ ] 无 P0 Bug
- [ ] P1 Bug ≤ 3 个（有修复计划）
- [ ] 所有 P0/P1 已修复通过验证
- [ ] 性能基准未下降
- [ ] 测试通过单已签收

## 阶段6：发布

- [ ] 所有文件存在且路径正确
- [ ] index.html 可正常打开
- [ ] 所有 JS 文件无语法错误
- [ ] CSS 加载完整
- [ ] 版本号已更新
- [ ] 发布说明已编写
- [ ] 上一版本已归档

## 阶段7：复盘

- [ ] 复盘会已召开（全员）
- [ ] 经验和教训已记录
- [ ] 改进计划已制定
- [ ] 技能更新提案已提交
