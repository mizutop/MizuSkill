# 复牌 (Retrospective)

## 概述

版本发布后的系统化复盘，识别改进点，推动工作室持续进化。

## 使用场景

- 版本发布完成后
- 项目里程碑到达时
- 周期性质量审计

## 复盘流程

### 1. 数据收集
- 各角色交付物完成情况
- Bug 统计（P0/P1/P2 分布）
- 计划 vs 实际工期对比
- 技能遵从报告

### 2. 四象限分析

| 做得好 (Keep) | 做得差 (Problem) |
|--------------|-----------------|
| {继续保持的} | {需要改进的} |

| 新想法 (Try) | 疑问 (Question) |
|-------------|----------------|
| {下次尝试的} | {未解决的问题} |

### 3. 改进计划

| 问题 | 改进措施 | 负责角色 | 预计版本 |
|------|---------|---------|---------|
| | | | |

### 4. 技能更新评估
- 现有技能是否需要更新？
- 是否需要新增技能？
- 是否有技能不再适用？

## 输出

```markdown
## 复盘报告

版本: {版本号}
日期: {YYYY-MM-DD}
参与角色: [全员]

### 数据摘要
- 计划故事点: {N}
- 完成故事点: {N}
- Bug 总数: {N} (P0: N, P1: N, P2: N)

### 改进计划
...
```
