# 标准工作流
> Mizuof 工作室 v3.0 ─ 4 种标准开发工作流

---

## 工作流1：新功能开发 (完整流水线)

```
阶段0 构思 → 阶段1 规划 → 阶段2 设计 → 阶段3 交互+视觉 → 阶段4 开发TDD → 阶段5 质量审计 → 阶段6 发布 → 阶段7 复盘
```

适用：全新功能从零到发布

---

## 工作流2：Bug 修复 (简化流水线)

```
阶段4 开发TDD → 阶段5 质量审计 → 阶段6 发布
```

适用：已有功能的缺陷修复

### 差异点
- 跳过构思/规划/设计/交互视觉阶段
- Developer 直接在修复分支上工作
- QA Guardian 聚焦回归测试

---

## 工作流3：UI 调整 (设计线)

```
阶段3 交互+视觉 → 阶段4 开发TDD → 阶段5 质量审计
```

适用：已有功能的界面优化，不涉及逻辑变更

### 差异点
- 跳过构思/规划/设计阶段
- UI Designer 输出增量规范
- Animator 输出增量动画

---

## 工作流4：数值/平衡性调整 (数据线)

```
阶段2 设计 → 阶段4 开发TDD → 阶段5 质量审计
```

适用：数值公式调整、平衡性优化

### 差异点
- Designer 直接输出数值调整表
- Developer 仅修改配置数据
- Test Architect 验证边界条件

---

## 工作流选择规则

| 场景 | 推荐工作流 | 预计流转角色数 |
|------|----------|-------------|
| 全新产品/功能 | 工作流1（完整） | 15 |
| 已有功能 Bug | 工作流2（简化） | 4 |
| UI/UX 优化 | 工作流3（设计线） | 6 |
| 数值平衡调整 | 工作流4（数据线） | 4 |
| 紧急线上故障 | 工作流2，跳过质量审计阶段 | 3 |
