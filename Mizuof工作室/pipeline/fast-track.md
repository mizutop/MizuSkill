# 快速通道流程

> 轻量级路径 ─ 跳过阶段 0-3，Bug修复/UI微调/数值调整/P0热修复

---

## 触发条件

满足**任意一条**即可启动快速通道：

| 条件 | 判定标准 |
|------|---------|
| Bug 修复 | 改动 ≤ 3 个文件 |
| UI 微调 | 仅 CSS 变更 |
| 数值调整 | 仅数据表常量变更 |
| P0 热修复 | Producer 判定为紧急 |

启动审批人：**Producer**

---

## 流程

```
快速启动 → 阶段4 开发 → 阶段5 质量审计 → 阶段6 发布

  ↓            ↓              ↓              ↓
Producer    Developer     QA Guardian     DevOps
批准       TDD→修复       回归→验证        标签→部署
```

**被跳过的阶段**（0-3）不需要重新执行，但修复的内容必须在修复说明中明确描述。

---

## 与其他通道的区别

| 维度 | 标准通道 | 快速通道 |
|------|---------|---------|
| 阶段数 | 8 | 4（启动+4+5+6） |
| 审批 | 每阶段质量门 | Producer 一次性审批 |
| 角色 | 15 角色全部按需 | 仅 Producer/Developer/QA/DevOps |
| 耗时 | 完整周期 | 1-2 轮即可发布 |

---

## 限制

- 快速通道一个月内使用不超过 **5 次**
- 连续 3 次快速通道 → 触发 Meta-Guardian 审计
- P0 热修复不占用快速通道配额
