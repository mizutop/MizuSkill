# 各阶段检查清单
> Mizuof 工作室 v2.0 ─ 15 角色 × 8 阶段检查项

---

## 阶段0：构思 ─ Producer 自检

- [ ] 1. 用户的核心需求是什么？
- [ ] 2. MVP 范围是否明确？（最简可行产品）
- [ ] 3. 目标用户画像是否清晰？
- [ ] 4. 核心差异化是什么？（为什么用户要选这个？）
- [ ] 5. 不做的范围（scope exclusion）是否明确？

---

## 阶段1：规划 ─ 全员校验

### Producer
- [ ] 核心循环是否清晰？
- [ ] 功能列表是否按 P0/P1/P2 分级？
- [ ] 里程碑是否有具体日期或版本号？
- [ ] 每个角色是否明确自己的交付物？
- [ ] 是否有明确的"完成"定义（DoD）？

### Architect
- [ ] 技术决策是否已记录 ADR？
- [ ] 系统边界是否已定义？
- [ ] 关键风险是否已识别？

### Director
- [ ] 执行计划是否已拆解为原子任务？
- [ ] 每个任务是否有明确的负责角色？

---

## 阶段2：设计 ─ Designer 18 项核对

### 数据结构
- [ ] 所有实体有完整的 JSON Schema 定义
- [ ] 所有字段有类型标注和默认值
- [ ] 嵌套深度不超过 3 层
- [ ] 数组字段有最大长度限制

### 数值公式
- [ ] 每个公式有推导过程说明
- [ ] 每个系数有选择理由
- [ ] 公式代入验证通过（至少 3 组数据点）
- [ ] 极端值测试通过（边界值验证）

### 边界条件
- [ ] 零值处理已定义
- [ ] 负数保护已实现
- [ ] 溢出保护（max safe integer）
- [ ] 空值/null 保护已定义

### 系统耦合
- [ ] 所有跨模块数据依赖已标注
- [ ] 事件触发链已绘制
- [ ] 循环依赖已排查

### 文档质量
- [ ] 所有表格有单位标注
- [ ] TODO 项有负责人和预计解决日期
- [ ] 与其他设计文档的交叉引用正确

---

## 阶段3：交互+视觉 ─ 全覆盖检查

### Interaction Designer
- [ ] 所有状态已覆盖：正常/空/加载/错误/禁用/极限
- [ ] 操作路径有最短路径和备选路径
- [ ] 异常退出路径已定义
- [ ] 交互反馈时延已标注

### UI Designer ─ 15 项审计
- [ ] 1. 色彩一致性
- [ ] 2. 间距对齐（4px/8px 网格）
- [ ] 3. 字体层级 ≤4 级
- [ ] 4. 可点击区域 ≥44px
- [ ] 5. 对比度 ≥4.5:1
- [ ] 6. 文字截断策略
- [ ] 7. 空状态
- [ ] 8. 加载态
- [ ] 9. 错误态
- [ ] 10. 焦点态
- [ ] 11. 按压态
- [ ] 12. 禁用态
- [ ] 13. 图标规范
- [ ] 14. Buff 图标
- [ ] 15. 装备视觉层级

### Animator
- [ ] 动画方案已标注性能预算
- [ ] 提供了 `prefers-reduced-motion` 降级方案
- [ ] GSAP 动画已优化（≤8 同时运行）

## 阶段4：开发 TDD ─ Developer 24 项自检

详见 `roles/developer.md` 和 `AI编程规范.md`。

关键检查项：
- [ ] 所有模块暴露 init/destroy 接口
- [ ] destroy 中解绑所有事件和定时器
- [ ] 所有外部输入有空值保护
- [ ] 所有 DOM 查询有返回值检查
- [ ] 模块间通过 Events 通信
- [ ] Magic Number 已替换为命名常量
- [ ] TODO 格式为 `// TODO(role): description`
- [ ] 遵循 `AI编程规范.md` 全部要求

- [ ] 游戏能正常启动
- [ ] 核心循环 1 次完整走通

- [ ] 所有按钮有响应（无死按钮）
- [ ] 弹窗能正常打开和关闭
- [ ] 页面切换无残留 DOM
- [ ] localStorage 无键名冲突
- [ ] 浏览器 Console 无红色报错
- [ ] 性能基准未下降
- [ ] 代码质量审计通过（遵循编码规范）
