# 流水线阶段定义（8 阶段）
> Mizuof 工作室 v3.0 ─ 从想法到完整生产价值完成品

---

## 阶段全景

```
阶段0          阶段1          阶段2          阶段3          阶段4          阶段5          阶段6          阶段7
 构思       →   规划       →   设计       →   交互+视觉   →   开发TDD   →   质量审计   →   发布       →   复盘
 ┃              ┃              ┃              ┃              ┃              ┃              ┃              ┃
Producer      Architect      Designer      Interaction    Developer      Test Arch.     DevOps        Meta-Guard
+Director                   +UI/UX/VA     +Animator      +QA Guardian   +QA Guardian                  +全员
 ┃              ┃              ┃              ┃              ┃              ┃              ┃              ┃
▼              ▼              ▼              ▼              ▼              ▼              ▼              ▼
想法收敛     技术方案      GDD+规范      UI+动画      可运行产品    审计报告      版本发布      改进计划
```

---

## 阶段0：构思 (Ideation)

| 项目 | 内容 |
|------|------|
| **负责** | Producer |
| **核心活动** | 用户意图探索 → `brainstorming` skill → 需求收敛 → MVP 范围定义 |
| **准入条件** | 用户有一个想法/需求 |
| **准出条件** | 收敛后的需求 + MVP 范围 |
| **输出物** | 需求收敛记录、MVP 范围定义 |

---

## 阶段1：规划 (Planning)

| 项目 | 内容 |
|------|------|
| **负责** | Producer + Architect + Director |
| **核心活动** | 编写 PRD → 技术方案 → ADR → 优先级排序 → 任务拆解 → 启动会 |
| **准入条件** | 需求已收敛 |
| **准出条件** | PRD 签收 + 启动会完成 + 执行计划就绪 |
| **评审节点** | 启动会（全员参加） |
| **输出物** | PRD、ADR、系统设计文档、执行计划、里程碑计划 |

---

## 阶段2：设计 (Design)

| 项目 | 内容 |
|------|------|
| **负责** | Designer |
| **核心活动** | 阅读 PRD → 设计数据结构和逻辑 → 数值公式推导 → 数值表编制 |
| **准入条件** | PRD + 系统设计文档签收 |
| **准出条件** | GDD 完成 + 设计评审通过 |
| **评审节点** | 设计评审会（Producer + Designer + Interaction + UI） |
| **输出物** | GDD、数据表定义、数值公式验证报告 |

---

## 阶段3：交互 + 视觉 (Interaction & Visual)

| 项目 | 内容 |
|------|------|
| **负责** | Interaction Designer → UI Designer → Visual Architect → Animator |
| **核心活动** | 交互流程 → 状态覆盖 → UI 规范 → 品牌视觉 → 动画方案 |
| **准入条件** | 设计评审通过 |
| **准出条件** | UI 规范签收 + UX 评估通过 + 动画方案就绪 |
| **评审节点** | UI/UX 评审会 |
| **输出物** | 交互流程图、UI 组件规范、视觉系统、GSAP 动画方案 |

---

## 阶段4：开发 TDD (Development TDD)

| 项目 | 内容 |
|------|------|
| **负责** | Developer + QA Guardian |
| **核心活动** | TDD 循环 → 代码实现 → 24 项自检 → 代码审查 → 提测 |
| **准入条件** | UI/UX 冻结签字 |
| **准出条件** | 代码完成 + 24 项自检通过 + 代码审查通过 |
| **评审节点** | 代码审查 |
| **输出物** | 可运行代码、模块接口文档、提测单 |

---

## 阶段5：质量审计 (Quality Audit)

| 项目 | 内容 |
|------|------|
| **负责** | Test Architect + QA Guardian |
| **核心活动** | 自动测试执行 → 功能测试 → 边界测试 → 回归测试 → 性能基准 → 代码质量审查 |
| **准入条件** | 提测单签收 + 24 项自检通过 |
| **准出条件** | 所有 P0/P1 Bug 已修复 + 测试通过单签收 |
| **评审节点** | 测试评审会（QA + Developer + Producer） |
| **输出物** | 测试报告、Bug 报告、性能基准报告、质量审计报告 |

---

## 阶段6：发布 (Release)

| 项目 | 内容 |
|------|------|
| **负责** | DevOps |
| **核心活动** | 文件完整性验证 → 打版本标签 → 编写发布说明 → 部署验证 → 归档 |
| **准入条件** | 测试通过单签收 |
| **准出条件** | 版本标签已打 + 发布说明已编写 + 部署验证通过 |
| **输出物** | 版本标签、发布说明、部署验证报告 |

---

## 阶段7：复盘 (Retrospective)

| 项目 | 内容 |
|------|------|
| **负责** | Meta-Guardian + 全员 |
| **核心活动** | 数据分析 → 经验教训 → 问题改进 → 技能更新提案 |
| **准入条件** | 版本发布完成 |
| **准出条件** | 改进计划 + 技能更新建议 |
| **评审节点** | 复盘会（全员） |
| **输出物** | 复盘报告、改进计划、技能更新提案 |

---

## 阶段流转规则

1. **禁止跳阶段** ─ 任何阶段不得跳过
2. **禁止并行越界** ─ Interaction 和 UI 可部分重叠，但 UI 不得在 Interaction 未冻结时开始
3. **驳回机制** ─ 下游发现上游输出不满足准入条件时，有权驳回并附具体问题清单
4. **冻结后变更** ─ 设计冻结后的任何修改须走 `Change Request` → Producer 审批
